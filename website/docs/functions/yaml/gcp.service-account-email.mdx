<Intro>
The `!gcp.service_account_email` YAML function retrieves the email address of the GCP service account
associated with the current caller identity.
</Intro>

## Usage

The `!gcp.service_account_email` function takes no parameters:

```yaml
  # Get the service account email of the current caller identity
  service_account: !gcp.service_account_email
```

## Arguments

This function takes no arguments. It uses GCP credentials from the environment or the
Atmos authentication context if configured.

## How It Works

When processing the `!gcp.service_account_email` YAML function, Atmos resolves the service account
email using the following priority order:

1. **Atmos GCP Auth Context** (highest priority) - If an active GCP identity is configured
   with a service account email, Atmos uses it directly.

2. **Application Default Credentials (ADC)** - If ADC is configured (e.g., via a service account
   key file or Workload Identity Federation), Atmos queries the OAuth2 tokeninfo API to discover
   the email of the credential's owner.

:::note ADC Email Resolution
Email discovery via ADC requires a valid access token. In some ADC configurations (e.g., user credentials
from `gcloud auth application-default login`), the token is associated with a user account rather than
a service account. In that case, the email reflects the user, not a service account.
:::

:::note Atmos Auth Integration
When using [Atmos Authentication](/cli/commands/auth/usage) with a GCP identity, the service account email
is typically available directly from the auth context, avoiding the ADC lookup entirely.
:::

## Caching

The `!gcp.service_account_email` function shares its cache with `!gcp.project_id`,
since both are resolved in a single identity lookup. Results are cached in memory for the duration
of the CLI invocation:

- Multiple uses of `!gcp.service_account_email` in the same command only make one identity resolution call
- Different authentication contexts get separate cache entries
- Each new CLI command starts with a fresh cache

:::

## Error Handling

If the function cannot resolve a service account email, it returns an empty string rather than failing.
Service account email is optional — unlike project ID, a missing email does not cause an error.

If the entire identity lookup fails (e.g., no project ID found), the function returns an error.

## Considerations

- **Email may be empty** - If no service account email is configured in any source, returns an empty string
- **ADC email discovery** - Requires a valid access token; may return a user email rather than service account email
- **Performance** - Shares the cache with `!gcp.project_id`
- **Explicit configuration preferred** - For predictable behavior, configure the email in the Atmos auth context
